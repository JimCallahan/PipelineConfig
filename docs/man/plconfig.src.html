Name

plconfig - the Pipeline Configuration Tool

Synopsis

plconfig [options]

plconfig --help
plconfig --html-help
plconfig --version
plconfig --release-date
plconfig --copyright

Description

This program is used by system administrators to generate a Pipeline configuration profile and license key for their site.  See the Global Options section for details about the site specific parameters which can be configured.

Before You Start

Before running this program, a new user and user group need to be created at your site for running Pipeline.  By default this user is called (pipeline), but another name can be specified with the --pipeline-user option. The plconfig(1) program must be run as the (pipeline) user.  This user will eventually be used to run various daemon processes and other privileged programs which make up the Pipeline system.  The (pipeline) user should be able to login to all hosts which will run Pipeline programs with an identical user and group ID.  Once a site configuration profile has been generated using plconfig(1), the user ID and group ID for this user should not be modified.

Configuring Pipeline

The set of options which must be specified are described in the Required Options section.  The rest of the options are not required and the configuration parameters they control are either auto-detected by plconfig(1) or given reasonable default values.  The choices for all configuration parameters will be printed by plconfig(1) when run. 

When run, plconfig(1) writes (or overwrites) the following three files:

plconfig/temerity-software.key

This file contains the license key required by Pipeline for your site.  This file should be zealously protected as it represents confidential information covered by the Temerity Software License Agreement.

Once Pipeline is installed, this file must be copied to the following location before Pipeline programs will start:

root-install-directory/pipeline-version-date/plconfig/temerity-software.key

Where (root-install-directory) is the directory specified by the --root-dir option and (pipeline-version-date) is the name of the pipeline distribution built for your site (see below).

 plconfig/customer-profile

This file contains the site specific configuration profile data which will be used build the custom Pipeline distribution for your site.  The contents of this file will need to be pasted into the Site Profile section of the online License Application Form.

plconfig/plconfig-settings.txt

This is a plain text file similar to the configuration parameters printed by plconfig(1).  This file is not required either to run Pipeline or as part of the license application process.  Its provided solely to document the configuration settings used to generating the other two files for your future reference.

Once you've decided on a configuration, you should run plconfig(1) one last time adding the --lock option the rest of your option choices. This option make the above three file read-only to prevent any accidental destruction at a later date by running plconfig(1) again.

Obtaining and Installing Pipeline

Once you have successfully completed the configuration process above, a distribution of Pipeline configured for your site may be obtained by completing the License Application Form.  Pipeline is distributed as a set of RPMs which are customized for your site packaged as a self extracting shell archive.  Once your license application as been processed, you will be sent an email containing the download location for this distribution file. 

The name of the distribution file will have the following format:

pipeline-customer-version-date.bin

Where (customer) is a short name assigned by Temerity Software for your site, (version) is the Pipeline release version number and (date) is the date that the distribution was built.   For example, a distribution of Pipeline 1.8.3 build for hypothetical Studio X on October 18, 2004 would be named:

pipeline-studiox-1.8.3-041018.bin

When executed, this example shell archive would create the following RPMs:

pipeline-studiox-1.8.3-041018/pipeline-studiox-1.8.3-041018.i686.rpm
pipeline-studiox-1.8.3-041018/plmaster-studiox-1.8.3-041018.i686.rpm
pipeline-studiox-1.8.3-041018/plqueumgr-studiox-1.8.3-041018.i686.rpm
pipeline-studiox-1.8.3-041018/plnotify-studiox-1.8.3-041018.src.rpm

The "pipeline" RPM contains the binaries which make up distribution and can be installed on any host which can write to the directories specified by the --root-dir and --prod-dir options.  Since the installation directory is located on a network accessable file system, this RPM only needs to be installed once for the site.

The "plmaster" RPM contains the directory structure and config files needed by the Pipeline master server daemon plmaster(1) and must be installed on the host specified by the --master-host option.  All files will be installed under the directory specified by the --node-dir option.

The "plqueuemgr" RPM contains the directory structure and config files needed by the Pipeline queue management daemon plmaster(1) and must be installed on the host specified by the --queue-host option.  All files will be installed under the directory specified by the --queue-dir option.

The "plnotify" source RPM is not normally needed, but is provided for site's which have unsual system configurations of file server host (see --file-host) which need to rebuild the native C++ libraries and binaries of Pipeline from sources.

System Prerequisites

Pipeline requires that the master server, file server and any hosts which be part of the Pipeline job execution queue run the Linux (kernel version 2.4 or later) operating system.  Pipeline has been developed and tested on a Red Hat 9.0 environment.  Support for other flavors of Linux or UNIX in general is possible but has not been tested.

In order to enable the full functionality of the Pipeline command-line and graphical client programs, they must also be run on a Linux host.  However, these client programs are written entirely in Java and may be run on any host which has a functional Java VM with some loss of functionality.  The biggest limitation is that editor/viewer programs cannot be launched by non-Linux Pipeline clients.  Otherwise, all other Pipeline operations are functional on non-Linux machines.

Finally, both Pipeline client and server programs are heavily dependent on Java.  Even the plconfig(1) tool depends on at least the Java Runtime Environment.

The existence and locations of the following Java prerequisites are checked by plconfig(1) when run and can be customized by the --class-path and --library-path options:

Java Runtime Environment
J2SE (5.0 or later)

Java Cryptography Extension
Unlimited Strength Jurisdiction Policy Files

Java Advanced Imaging
JAI (1.1.2 or later)
JAI Image I/O API (1.0 or later)

Java 3D Library
Java 3D API (1.3.1 or later)

Information Options

The following options provide information about plconfig(1) and then immediately exit:
--help
Prints a brief usage message and exits.
--html-help
Opens this page using the mozilla(1) web browser. 
--version
Prints the version number of this software and exits.
--release-date
Prints date that this software was released and exits.
--copyright

Prints copyright notice for this software and exits.

Global Options

Graphical Mode Option

--graphical

Display a set of graphical dialogs which will walk you through the configuration process. If this option is given, it must be the only option to plconfig(1).  If this option is not given, plconfig(1) will operate in non-graphical mode.

Required Options

If run in non-graphical mode, the following set of options must be specified each time plconfig(1) is run. These options control the minimal set of configuration parameters required for any site.

--root-dir=dir

The path to the directory (dir) under which the top-level directory of the Pipeline installation will reside. This directory should reside on a network accessible file system such that all machines which will run Pipeline related programs can access the directory as (dir).  The top-level directory of the Pipeline installtion will have a name matching the Pipeline distribution.

--evaluation
--annual
--perpetual

Specifies the time period the Pipeline license will remain valid.  An evaluation license is valid for 30-days. An annual license is valid for one year.  A perpetual license is valid forever.  One and only one of these options must be given.

Non-Required Options

If run in non-graphical mode, the following set of options are not required but may be used to customize the configuration parameters for your site which would otherwise be auto-detected or given default values.

--pipeline-user=name

The name of the user which will run Pipeline daemon processes and have special permission to perform Pipeline administrative operations. 

If not specified, the pipeline user will default to (pipeline).

--host-ids=filename

Specifies the name of the file (filename) containing the host IDs for the Pipeline server hosts specified by the --master-host, --queue-host and --file-host options.  This host ID file is generated by running plid(1) on each of the server hosts.  The file can either be constructed manually by appending the output from each invocation of plid(1) or automatically by using the --append option to plid(1).


--home-dir=dir

The absolute path to the parent directory of all user home directories.  If not specified, this location is derived from the environment under which plconfig(1) is run or (/home) if the location cannot be determined from the environment.

--temp-dir=dir

The absolute path to the directory (dir) used to write various temporary files generated by Pipeline programs.  For optimal performance, this directory should reside on a local file system.

If not specified, the temporary directory will default to (/usr/tmp).


--master-host=host

The host name of the machine (host) which will run the Pipeline master server daemon plmaster(1).  This daemon should run on a host which can access the root node directory (see --node-dir) on a local file system.  The master server daemon is responsible for managing all Pipeline node operations including dispatching jobs.  For optimal performance, the host which runs this daemon should be dedicated and have a fast network connection to the host running the Pipeline file management plfilemgr(1) daemon (see --file-host). 

Note that a host ID for this host must be generate using plid(1) and placed in the host IDs file passed to the --host-ids option.

If not specified, the master host will default to (localhost).

--master-port=num

The number of the network port (num) listened to by the Pipeline master server daemon plmaster(1). 

If not specified, the port number will default to (53135).

--node-dir=dir

The absolute path to the root directory under which the persistent copies of Pipeline nodes are stored.  This directory should reside on a local file system on the host which will run plmaster(1)  (see --master-host). 

If not specified, the node directory will default to (/usr/share/pipeline).



--file-host=host

The host name of the machine (host) which will run the Pipeline file management daemon plfilemgr(1).  For optimal performance, this daemon should be run on the same host which acts as the network file server where production data files will reside (see --prod-dir).  If local file acess is not possible or desireable, the plfilemgr(1) daemon should be run on a dedicated host which has the fastest possible network access to the production directories.

Note that a host ID for this host must be generate using plid(1) and placed in the host IDs file passed to the --host-ids option.

If not specified, the file host will default to (localhost).

--file-port=num

The number of the network port (num) listened to by the Pipeline file management daemon plfilemgr(1). 

If not specified, the port number will default to (53136).

--prod-dir=dir

The absolute path to the root directory under which all production data files will reside.  In other words, the location where artists will save their work.  This root production directory should reside on a network accessible file system such that all machines which will run Pipeline related programs can access the directory as (dir).

For optimal performance this directory should reside on the same host which runs the plfilemgr(1) daemon (see --file-host) although this is not required.

If not specified, the production directory will default to (/base/prod).


--queue-host=host

The host name of the machine (host) which will run the Pipeline queue management daemon plqueuemgr(1).  This daemon assigns jobs from the queue to the hosts running the job execution pljobmgr(1) daemons.  It also provides information about the status of these jobs to both the Pipeline master server plmaster(1) and plui(1) client programs.

Note that a host ID for this host must be generate using plid(1) and placed in the host IDs file passed to the --host-ids option.

If not specified, the file host will default to (localhost).

--queue-port=num
The number of the network port (num) listened to by the Pipeline queue management daemon plqueuemgr(1).

If not specified, the port number will default to (53139).

--job-port=num

The number of the network port (num) listened to by the Pipeline job execution pljobmgr(1) daemons.  Each host which executes jobs on behalf of the Pipeline job queue will run this daemon.

If not specified, the port number will default to (53140).

--queue-dir=dir
The absolute path to the root directory under which all queue related data files will reside.  This directory should reside on a local file system on the host which will run the Pipeline queue server daemon plqueuemgr(1) (see --queue-host). 

If not specified, the node directory will default to (/usr/share/pipeline).



--class-path=path

An additional set of colon (:) seperated directories (path) to search for the JAR files for the Java prerequisites described in the System Prerequisites section.  This (path) is appended to the normal class search path for the Java Runtime Environment. Normally this option is not required as the JAR files for JAI and Java 3D are usually installed in the same location as the JRE.  This option is provided to support non-standard installations of these support packages.

--library-path=path

An additional set of (:) seperated directories (path) to search for the native shared object libraries for the Java prerequisites described in the System Prerequisites section.  This path (path) is appended to the normal libaray search path for the Java Runtime Environment. Normally this option is not required as the native libraries for JAI and Java 3D are usually installed in the same location as the JRE.  This option is provided to support non-standard installations of these support packages.

If not specified, the value of the LD_LIBRARY_PATH environmental variable is searched instead for native libraries as well as the standard JRE locations.
