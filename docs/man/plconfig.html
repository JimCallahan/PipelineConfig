Name

plconfig - the Pipeline configuration and license generator program

Synopsis

plconfig [options]

plconfig --graphical

plconfig --help
plconfig --html-help
plconfig --version
plconfig --release-date
plconfig --copyright

Description

This program is used by system administrators to generate a Pipeline configuration profile and license key for their site.  See the Global Options section for details about the site specific parameters which can be configured.

Before You Start

Before running this program, a new user called (pipeline) who's user group is also called (pipeline) needs to be created at your site.  The plconfig(1) program must be run as the (pipeline) user.  This user will eventually be used to run various daemon processes and other privileged programs which make up the Pipeline system.  The (pipeline) user should be able to login to all hosts which will run Pipeline programs with an identical user and group ID.  Once a site configuration profile has been generated using plconfig(1), the user ID and group ID for this user should not be modified.

In addition, the root installation directory (see --root-dir) for Pipeline needs to be created and have its file access permissions and ownership set before running plconfig(1) exactly as follows:

> mkdir root-install-directory
> chown root.pipeline root-install-directory
> chmod 775 root-install-directory

Where the (root-install-directory) is replaced with the location chosen for the Pipeline installation for your site.  Once these steps have been taken, you are ready to configure PIpeline for your site.

Configuring Pipeline

Once the steps described above, its a relatively simple matter to generate the Pipeline configuration and license key. 

If run in graphical mode, plconfig(1) only requires the --graphical option and will display a set of dialogs which will walk you through the configuration process.

Otherwise, plconfig(1) is run in command-line mode and requires a core set of options described in the Required Options section.  The rest of the options are not required and the configuration parameters they control are either auto-detected by plconfig(1) or given reasonable default values.  The choices for all configuration parameters will be printed by plconfig(1) when run. 

Regardless of whether plconfig(1) is run in graphical or command-line mode, it writes (or overwrites) the following three files under the root installation directory (see --root-dir):

root-install-directory/pipeline-license.key

This file contains the license key for Pipeline for your site.  This file is created with file access permissions which make it readable only by the (pipeline) user. This file should be zealously protected as it represents confidential information covered by the Pipeline License Agreement. Once Pipeline is installed, this file will be required in order to run the programs which make up the Pipeline.

root-install-directory/pipeline.profile

This file contains the site specific configuration profile data which will be used build the custom Pipeline distribution for your site.  The contents of this file will need to be pasted into the Site Profile section of the online Pipeline License Application Form.

root-install-directory/pipeline.config

This is a plain text file similar to the configuration parameters printed by plconfig(1).  This file is not required either to run Pipeline or as part of the license application process.  Its provided solely for your future reference of the configuration settings you chose when generating the other two files.

Once you've decided on a configuration, you should run plconfig(1) one last time adding the --lock option the rest of your option choices. This option make the above three file read-only to prevent any accidental destruction at a later date by running plconfig(1) again.

Obtaining and Installing Pipeline

Once you have successfully completed the configuration process above, a distribution of Pipeline  for your site may be obtained by completing the Pipeline License Application Form.  Pipeline is distributed as an RPM which are customized for your site.  Once your license application as been processed, you will be sent an email containing the download location for this custom RPM for your site. 

System Prerequisites

Pipeline requires that the master server, file server and any hosts which be part of the Pipeline job execution queue run the Linux (kernel version 2.4 or later) operating system.  Support for other flavors of UNIX are possible as long as they support the same file system related system calls as Linux.

In order to enable the full functionality of the Pipeline command-line and graphical client programs, they must also be run on a Linux host.  However, these client programs are written entirely in Java and may be run on any host which has a functional Java VM with some loss of functionality.  The biggest limitation is that editor and viewer programs cannot be launched by non-Linux Pipeline clients.  Otherwise, all other Pipeline operations are functional on non-Linux machines.

Finally, both Pipeline client and server programs are heavily dependent on Java.  Even the plconfig(1) tool depends on at least a minimal Java Runtime Environment.  The existence and locations of the following Java prerequisites are checked by plconfig(1) when run and can be customized by plconfig(1) options:

Java Runtime Environment
J2SE (1.5.0 or later)

Java Advanced Imaging
JAI (1.1.2 or later)
JAI Image I/O API (1.0 or later)

Java 3D Library
Java 3D API (1.3.1 or later)

Information Options

The following options provide information about plconfig(1) and then immediately exit:
--help
Prints a brief usage message and exits.
--html-help
Opens this page using the mozilla(1) web browser. 
--version
Prints the version number of this software and exits.
--release-date
Prints date that this software was released and exits.
--copyright

Prints copyright notice for this software and exits.

Global Options

Graphical Mode Option

--graphical

Display a set of graphical dialogs which will walk you through the configuration process. If this option is given, it must be the only option to plconfig(1).  If this option is not given, plconfig(1) will operate in non-graphical mode.

Required Options

If run in non-graphical mode, the following set of options must be specified each time plconfig(1) is run. These options control the minimal set of configuration parameters required for any site.

--root-dir=dir

The path to the root installation directory (dir) of Pipeline.  All programs and data files which make up Pipeline will live under this directory when Pipeline is installed.  This directory should reside on a network accessible file system such that all machines which will run Pipeline related programs can access the directory as (dir).

This directory must exist and be writable by the "pipeline" user at the time plconfig(1) is run.

--evaluation
--annual
--perpetual

Specifies the time period the Pipeline license will remain valid.  An evaluation license is valid for 30-days. An annual license is valid for one year.  A perpetual license is valid forever.  One and only one of these options must be given.

Non-Required Options

If run in non-graphical mode, the following set of options are not required but may be used to customize the configuration parameters for your site which would otherwise be auto-detected or given default values.

--domain=name

The fully qualified domain name of the site for which Pipeline is being configured. For example, (your-studio.com).  Pipeline licenses are always site-wide and specific to a particular internet domain.

If not specified, the domain name will be determined from the name of the machine running plconfig(1).

--home-dir=dir

The absolute path to the parent directory of all user home directories. 

If not specified, this location is derived from the environment under which plconfig(1) is run or (/home) if the location cannot be determined from the environment.

--toolset-dir=dir

 The absolute path to the root directory (dir) of the installed toolsets for the site.  Toolsets represent named shell environments under which various Pipeline editors and jobs are run.  A toolset is simply a shell script containing only environmental variable definitions.  The location chosen for toolsets normally is on a network accessible file system.  Alternatively, this directory may be located on a local file system on each machine which is kept consistent via some automated process such as rdist(1). In either case, this directory needs to be accessible from all machines which will run Pipeline related programs and have a consistent contents.

If not specified, the toolset directory will default to (/base/toolset).


--master-host=host

The host name of the machine (host) which will run the Pipeline master server plmaster(1) daemon.  This daemon should run on a host which can access the root node directory (see --node-dir) on a local file system.  The master server daemon is responsible for managing all Pipeline node operations including dispatching jobs.  For optimal performance, the host which runs this daemon should be dedicated and have a fast network connection to the host running the Pipeline file management plfilemgr(1) daemon (see --file-host). 

If not specified, the master host will default to (localhost).

--master-port=num

The number of the network port (num) listened to by the Pipeline master server plmaster(1) daemon. 

If not specified, the port number will default to (53135).

--node-dir=dir

The absolute path to the root directory under which the persistent copies of Pipeline nodes and node related information are stored.  This directory should reside on a local file system on the host which will run the Pipeline master server plmaster(1) daemon (see --master-host). 

If not specified, the node directory will default to (/usr/share/pipeline).



--file-host=host

The host name of the machine (host) which will run the Pipeline file management plfilemgr(1) daemon.  This daemon should be run on the same host which acts as the network file server where production data files will reside on your site for optimal performance (see --prod-dir).

If not specified, the file host will default to (localhost).

--file-port=num

The number of the network port (num) listened to by the Pipeline file management daemon plfilemgr(1). 

If not specified, the port number will default to (53136).

--prod-dir=dir

The absolute path to the root directory under which all production data files will reside.  In other words, the location where artists will save their work.  This root production directory should reside on a network accessible file system such that all machines which will run Pipeline related programs can access the directory as (dir). For optimal performance this directory should reside on the same host which runs the file management daemon plfilemgr(1) (see --file-host).

If not specified, the production directory will default to (/base/prod).



--class-path=path

An additional set of colon (:) seperated directories (path) to search for the Java prerequisites described in the System Prerequisites section.  This (path) is appended to the normal class search path for the Java Runtime Environment and the JAR file which make up the Pipeline distribution.  Normally this option is not required as the JAR files for JAI and Java 3D are usually installed in the same location as the JRE.  This option is provided to support non-standard installations of these support packages.



--lock

Makes the generated configuration files (pipeline-license.key, pipeline.profile and pipeline.config) read-only, to protect them from being overwritten by a later execution of plconfig(1).  It is strongly advised that plconfig(1) be run with this option once a final configuration has been decided on to prevent accidental damage to your Pipeline license key and site profile.
